**Goal**: Enable NPCs to possess deep, accurate knowledge of Ultima Online lore including history, locations, dungeons, characters, and events while maintaining high performance.

**Current State**: 47 lore entries, 5 sent to LLM per conversation (truncated to 150 chars)
**Target State**: 500+ lore entries, hierarchical compression, dynamic retrieval

## Phase 8: Implementation Strategy (Week 8-12)

### 8.1 Knowledge Boundary System Integration

**Core Principle**: NPCs should feel authentic by knowing what they *should* know and gracefully deferring when out of their depth.

**Expertise Levels**:
```csharp
public enum KnowledgeExpertise
{
    Expert,      // Deep knowledge, can explain at length
    Proficient,  // Working knowledge, practical understanding
    Aware,       // Basic facts only, knows it exists
    Ignorant     // Doesn't know, should refer elsewhere
}
```

**Domain-Based Knowledge Access**:
- Blacksmith: Expert in weapons/ores, Ignorant in magic → "Ask mages at Moonglow"
- Mage: Expert in spells/reagents, Ignorant in smithing → "Ask a blacksmith"
- Scholar: Expert in history, Proficient in most topics
- Guard: Expert in threats/dungeons, Proficient in local knowledge
- Commoner: Aware of basics, Ignorant in most specialties → "That's above my station"

**Benefits**:
- ✅ More immersive (NPCs feel realistic)
- ✅ Creates interconnected world (referrals to other NPCs)
- ✅ Better performance (smaller, focused knowledge per NPC)
- ✅ Encourages player exploration (seek different specialists)

### 8.2 Data Structure Design

**Hierarchical Lore Entry Format with Domain Tags:**
```json
{
  "id": "mondain",
  "title": "Mondain the Wizard",
  "category": "NPC",
  "subcategory": "Villain",
  "era": "Ancient",
  "importance": 10,

  "summary": "Evil wizard defeated by the Stranger, created Gem of Immortality",

  "fullContent": "Mondain was the son of King Wolfgang and sought power above all else. He murdered his father to obtain the Sun Ruby, which he transformed into the Gem of Immortality...",

  "keyFacts": [
    "Murdered his father King Wolfgang",
    "Created the Gem of Immortality",
    "Enslaved Sosaria",
    "Defeated by the Stranger via time travel",
    "His death sundered the world"
  ],

  "relatedEntries": ["minax", "exodus", "gemofimmortality", "stranger", "sosaria"],

  "tags": ["mondain", "wizard", "villain", "gem", "immortality", "ancient"],

  "embeddings": [0.23, -0.45, 0.12, ...],

  "domainExpertise": {
    "Scholar": "Expert",      // Scholars know detailed history
    "Mage": "Expert",         // Mages know about famous wizards
    "Guard": "Aware",         // Guards know basic history
    "Blacksmith": "Aware",    // Blacksmiths know general history
    "Commoner": "Aware",      // Commoners know the legend
    "Merchant": "Aware",      // Merchants know basics
    "Healer": "Ignorant"      // Not relevant to healing
  },

  "referralTarget": {
    "primary": "Scholar",     // Best source for detailed info
    "secondary": "Mage"       // Alternative expert source
  }
}
```

**Weapon Entry Example (Blacksmith Expert Domain)**:
```json
{
  "id": "valorite_ore",
  "title": "Valorite Ore",
  "category": "Crafting",
  "subcategory": "Ore",
  "importance": 8,

  "summary": "Rarest ore, strongest metal, magic-resistant",

  "fullContent": "Valorite is the rarest and strongest ore in all of Britannia. It resists magic better than any other metal and holds enchantments beautifully. Found deep in dungeons like Shame and Destard. Requires tremendous heat (white-hot, not just red) and masterful hammer control to forge properly. One wrong strike can ruin the entire piece...",

  "domainExpertise": {
    "Blacksmith": "Expert",   // Can discuss in detail
    "Weaponsmith": "Expert",
    "Armorer": "Expert",
    "Miner": "Proficient",    // Knows where to find it
    "Guard": "Aware",         // Seen valorite weapons
    "Mage": "Aware",          // Knows magic resistance property
    "Scholar": "Proficient",  // Academic knowledge
    "Merchant": "Proficient", // Knows pricing/value
    "Commoner": "Ignorant"    // Never seen it
  },

  "referralTarget": {
    "primary": "Blacksmith",
    "secondary": "Weaponsmith"
  }
}
```

### 8.2 Compression System

**Tier 1: Ultra-Compact (Always Loaded)**
- 500+ entries compressed to 100-200 tokens total
- Format: "Mondain (villain, 1355), Britain (capital, Compassion), Destard (dragons)..."

**Tier 2: Category Summaries (Role-Specific)**
- 20-30 entries per role, 300-500 tokens
- Format: Full summaries but not full content

**Tier 3: Full Entries (On-Demand)**
- Retrieved when player asks specific questions
- Full details, unlimited length

### 8.3 Knowledge Retrieval Patterns

**Pattern 1: Proactive Loading**
```csharp
OnNPCSpawn():
  - Load Tier 1 (all compressed knowledge)
  - Load Tier 2 (role-specific summaries)
  - Cache in NPC memory
```

**Pattern 2: Dynamic Retrieval**
```csharp
OnPlayerQuestion():
  - Detect topic from question
  - Load Tier 3 (full entry) for that topic
  - Load related entries (graph traversal)
  - Send to LLM with full details
```

**Pattern 3: Predictive Caching**
```csharp
AfterResponse():
  - Analyze conversation context
  - Predict related topics player might ask
  - Pre-cache full entries for instant retrieval
```

### 8.4 Vector Embedding Strategy

**Generate embeddings for:**
- All lore entry titles
- All lore entry summaries
- All lore entry full content
- Player questions (on-the-fly)

**Semantic matching enables:**
- "Tell me about the king" → matches "Lord British", "Blackthorn", "Casca"
- "Dragons" → matches "Destard", "Ancient Wyrm", "Drakes"
- "Magic" → matches "Eight Circles", "Reagents", "Spellweaving"

### 8.5 Knowledge Graph

**Build relationships:**
```
Mondain → created → Gem of Immortality
Mondain → taught → Minax
Mondain + Minax → created → Exodus
Mondain → ruled → Sosaria
Mondain → defeated by → Stranger
```

**Enable queries like:**
- "Who created Exodus?" → Traverse graph to Mondain + Minax
- "What happened after Mondain fell?" → Traverse to Minax's revenge
- "Tell me about Britannia's enemies" → Find all villain nodes
